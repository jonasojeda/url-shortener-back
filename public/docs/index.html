Url

Display a listing of the resource.

This method handles a GET request to retrieve a list of URLs from the database. It allows filtering results based on optional search parameters, ordering results ascending or descending, and paginating the results.

Accepted query parameters:

Example request:
curl --request GET \
    --get "http://localhost/api/url?search=%22example.com%22.&order_desc=1&order_asc=1&unpaginated=1&page=1" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/url"
);

const params = {
    "search": ""example.com".",
    "order_desc": "1",
    "order_asc": "1",
    "unpaginated": "1",
    "page": "1",
};
Object.keys(params)
    .forEach(key => url.searchParams.append(key, params[key]));

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

Example response (200):


{
   "data": [
       {
           "id": 3,
           "url": "https://www.example.com/",
           "url_key": "g9fA6ac8",
           "short_url": "http://127.0.0.1:8000/g9fA6ac8"
       },
   ],
   "current_page": 1,
   "last_page": 1,
   "total": 2
}

Additional details:

- If the `unpaginated` parameter is true, all records will be returned without pagination.
- If both `order_asc` and `order_desc` are specified, only the last evaluated parameter will be applied.

Headers

Example: application/json

Example: application/json

Query Parameters

search   string  optional  

Optional. A search filter based on partial URL matches. Example: "example.com".

order_desc   boolean  optional  

Optional. Sorts the results in descending order by the created_at column. Default is true if not specified. Example: true

order_asc   boolean  optional  

Optional. Sorts the results in ascending order by the created_at column. If present, order_desc is not applied. Example: true

unpaginated   boolean  optional  

Optional. If present and true, it returns all results without pagination. Example: true

page   integer  optional  

Optional. The page number for pagination. Defaults to the first page if not specified. Example: 1

Create a Shortened URL

This method allows a user to submit a full URL and generates a shortened version of it. The provided URL is validated, ensuring it is a well-formed URL. A unique, random key is generated, and the original URL and its shortened key are stored in the database. If the provided URL is not valid, a validation error is returned.

Example request:
curl --request POST \
    "http://localhost/api/url" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"url\": \"https:\\/\\/www.google.com\"
}"

const url = new URL(
    "http://localhost/api/url"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "url": "https:\/\/www.google.com"
};

fetch(url, {
    method: "POST",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (201):


{
    "data": {
        "id": 4,
        "url": "https://www.example.com",
        "url_key": "Lm2qnRc1",
        "short_url": "http://127.0.0.1:8000/Lm2qnRc1"
    }
}

Example response (400):


{
    "status": false,
    "errors": [
        "The url format is invalid."
    ]
}

Retrieve the Original URL by its Shortened Key (url_key)

This method retrieves the original URL associated with a shortened key (url_key). It searches the database for the provided key, and if the key exists, returns the original URL. If the key is not found, an error message is returned indicating the URL was not found.

Example request:
curl --request GET \
    --get "http://localhost/api/url/error" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/url/error"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "GET",
    headers,
}).then(response => response.json());

Example response (200):


{
    "data": {
        "id": 1,
        "url": "https://example.com",
        "url_key": "abc12345",
        "created_at": "2024-01-01T12:00:00Z",
        "updated_at": "2024-01-01T12:00:00Z"
    }
}

Example response (404):


{
    "status": false,
    "message": "URL not found"
}

Update the specified resource in storage.

This method handles a PUT or PATCH request to update an existing URL in the database.

Example request:
curl --request PUT \
    "http://localhost/api/url/1" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json" \
    --data "{
    \"url\": \"\\\"https:\\/\\/www.google.com\\\".\"
}"

const url = new URL(
    "http://localhost/api/url/1"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

let body = {
    "url": "\"https:\/\/www.google.com\"."
};

fetch(url, {
    method: "PUT",
    headers,
    body: JSON.stringify(body),
}).then(response => response.json());

Example response (200):


{
   "data": {
       "id": 4,
       "url": "https://www.example.com",
       "url_key": "Lm2qnRc1",
       "short_url": "http://127.0.0.1:8000/Lm2qnRc1"
   }
}

Example JSON response on validation failure:

```json
{
  "status": false,
  "errors": [
    "The url field must be a valid URL."
  ]
}
```

Additional details:

- If the `url` parameter is not provided, the existing URL in the database remains unchanged.
- This method validates the incoming request data and returns a 400 response if validation fails.

Remove the specified resource from storage.

This method handles a DELETE request to remove a URL from the database.

Example request:

    "http://localhost/api/url/1" \
    --header "Content-Type: application/json" \
    --header "Accept: application/json"
const url = new URL(
    "http://localhost/api/url/1"
);

const headers = {
    "Content-Type": "application/json",
    "Accept": "application/json",
};

fetch(url, {
    method: "DELETE",
    headers,
}).then(response => response.json());

Example response (200):


{
   "data": {
       "id": 4,
       "url": "https://www.example.com",
       "url_key": "Lm2qnRc1",
       "short_url": "http://127.0.0.1:8000/Lm2qnRc1"
   }
}

Additional details:

- This method assumes the URL exists in the database. If the URL does not exist, it will return a 404 error by default.
- The response will have a status code of 200 (HTTP_OK) upon successful deletion.
